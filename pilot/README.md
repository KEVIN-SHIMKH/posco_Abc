# 후판 수주 문의 응답 챗봇 — 파일럿

`docs/기획서_후판수주문의챗봇.md` 설계를 mock 데이터로 검증하는 파일럿입니다.
실제 SME 데이터/규칙이 아직 없어 `data/*.xlsx`는 전부 가정한 mock 값입니다
(`docs/기획서_후판수주문의챗봇.md` §8 미결 사항 참고).

## 실행 방법

```bash
cd pilot
py -m venv .venv                     # 최초 1회
./.venv/Scripts/python -m pip install -r requirements.txt

cp .env.example .env
# .env 파일에 본인의 ANTHROPIC_API_KEY 입력 (직접 편집, 커밋 금지)

./.venv/Scripts/python app.py
# http://127.0.0.1:5000 접속
```

## 화면 구성

상단 "문의 정보 입력" 패널(출강목표 / 규격 / 열처리재 / 두께 / 폭 / 길이 + 추가 설명 텍스트)에
값을 채우고 "문의하기"를 누르면 아래 "대화" 로그에 응답이 스트리밍된다. 구조화 필드는
LLM이 자유 텍스트에서 추출한 값보다 항상 우선 적용된다 (사용자가 직접 입력한 값은 임의
추정이 아니므로). 추가 설명 없이 폼만 채워도 동작한다 — 이 경우 LLM 의도 분류 호출 자체를
건너뛰고 채워진 필드로 바로 라우팅한다.

## 확인 가능한 시나리오

1. **사이즈 기준 문의** — 규격 `AH36`, 두께 `25`, 폭 `2500`, 길이 `8000` 입력 후 문의하기
   → 경로 B(사이즈 기준 판정) 동작 확인
2. **열처리재 문의** — 위 입력에 "열처리재" 체크 → 판정 결과와 별도로 "열처리 조건은
   기준 데이터에 반영되어 있지 않음" 안내가 함께 나오는지 확인 (§8-12 관련)
3. **출강목표로 여재 조회** — 출강목표에 `C170150PG201` 입력 후 문의하기 (다른 필드 비움)
   → 경로 C, 해당 출강목표의 slab만 필터링되어 조회되는지 확인
4. **정형 데이터 조회** — 추가 설명에 "AH36 소재로 과거에 투입한 실적 있어? 여재slab도 있어?"
   입력 → 경로 C(실적/여재slab 조회 + 참고의견 문구) 동작 확인
5. **매칭 규칙 없는 사이즈** — 두께에 `300` 같은 극단값 입력 → "확인필요" 응답 확인 (임의 추정 안 함)
6. **소LOT 경로** — "테스트용 주문 컨텍스트"에서 "주문서 작성됨" 체크 + 수량 30 입력 후
   아무 문의나 전송 → LLM 호출 없이 즉시 링크 카드만 반환되는지 확인
7. **피드백 저장** — 응답 후 피드백 입력 → `logs/feedback.jsonl`에 누적되는지 확인

## 알려진 제약 (mock 파일럿 한계)

- `data/*.xlsx`는 실제 데이터가 아님 — 판정 결과를 업무에 사용하지 말 것
- 소LOT 판정 threshold(`SMALL_LOT_THRESHOLD_TON`)와 "주문서 작성 상태" 판별 기준은 SME 확인 전 임시값
- 대화 상태(세션별 `order_context` 누적, 기획서 §2.4)는 이번 파일럿에 아직 미구현 — 매 요청이
  독립적으로 처리됨
- 소LOT 에이전트 URL은 placeholder (`SMALL_LOT_AGENT_URL`)
